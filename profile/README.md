# 씨드데이 (Seeday)

> **3가지 기록을 통해 꾸준함과 습관을 만들어 보세요!**  
> 일상, 운동, 습관 기록을 통해 나만의 루틴을 만들고 성장하는 경험을 제공하는 서비스입니다.

---

## 🌱 씨드데이란?

씨드데이는 사용자의 **하루를 기록하고 돌아보는 경험**을 통해  
꾸준함과 좋은 습관을 만드는 것을 목표로 하는 서비스입니다.

- 하루의 감정과 일상을 기록하며 **나를 돌아보고**
- 운동 기록을 통해 **건강한 루틴을 만들고**
- 고치고 싶은 습관, 만들고 싶은 습관을 기록하며 **긍정적인 변화를 쌓아가는** 서비스입니다.

---

## ✏️ 씨드데이의 3가지 기록

### 1. 일상 기록
> 하루의 감정과 일상 기록을 통해 꾸준함과 나만의 루틴을 만들어 보세요.

- 오늘 있었던 일, 감정, 생각을 자유롭게 기록
- 일상의 작은 순간들을 모아 나만의 히스토리로 쌓기
- 시간이 지나도 다시 돌아볼 수 있는 나만의 기록 공간

### 2. 운동 기록
> 운동 기록을 통해 건강한 나를 만들고, 성취감과 운동 습관을 만들어 보세요.

- 오늘 한 운동과 시간을 간단하게 기록
- 누적 운동 기록을 통해 내가 얼마나 움직였는지 한눈에 확인
- 꾸준한 운동 루틴을 만들어 가며 성취감 경험

### 3. 습관 기록
> 고치고 싶은 습관이나 만들고 싶은 습관을 기록하고, 긍정적인 변화를 느껴보세요.

- 만들고 싶은 좋은 습관, 줄이고 싶은 나쁜 습관을 등록
- 매일 체크하면서 변화 과정을 시각적으로 확인
- 작은 변화가 쌓여 큰 변화를 만들어 가는 경험 제공

## 🏗 Seeday 조직 구조

Seeday 조직은 여러 플랫폼/역할별 리포지토리로 구성되어 있습니다.

- **[Android](https://github.com/Record-Management/Android)**
  - 씨드데이 안드로이드 앱
  - Kotlin, Multi-module 구조, Android Modern Architecture 기반

- **[IOS](https://github.com/Record-Management/IOS)**
  - 씨드데이 iOS 앱
  - Swift 기반 iOS 클라이언트

- **[Backend](https://github.com/Record-Management/Backend)**
  - 씨드데이 백엔드 서버
  - 씨드데이의 핵심 도메인/데이터를 담당하는 API 서버

- **[.github](https://github.com/Record-Management/.github)**
  - Seeday 조직 공용 설정 및 소개 리포지토리
  - 조직 프로필 및 템플릿 관리

## 🔧 기술 개요

플랫폼별로 세부 기술 스택은 리포지토리 README를 참고하면 좋고,  
조직 전체 관점에서 보면 대략 다음과 같은 기술을 사용하고 있습니다.

- **모바일 클라이언트**
  - Android: Kotlin, 멀티 모듈
  - iOS: Swift 기반 클라이언트

- **백엔드**
  - Java 기반 서버 (상세 스택은 Backend 리포지토리 참고)

- **공통**
  - GitHub Organizations & Repositories
  - 이슈/PR 기반 협업 프로세스
  - 환경에 따라 CI/CD 및 코드 스타일/컴벤션 적용(각 리포지토리별 상이)

> 자세한 기술 스택, 아키텍처, 의사결정 내용은 각 리포지토리의 README 및 문서를 참고해주세요.
